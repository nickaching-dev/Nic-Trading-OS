## Purpose

Build NicOS itself into a durable personal operating system that can support multiple domains without becoming messy or bloated.

## Vision

Turn NicOS into a second brain that helps Nic think better, track work clearly, and continuously improve decision quality across modules and projects.

## Goal

Design and evolve NicOS so it can support trading, projects, and future life domains with clean structure and useful workflows.

## Success Criteria

- The system remains easy to navigate
- Modules and projects stay clearly separated
- Knowledge flows back into permanent modules
- New features improve clarity rather than just adding activity

## Decisions

- Decision quality is more important than system complexity
- Modules store permanent knowledge
- Projects are temporary workspaces
- Betting is managed outside NicOS
- Meaningful sessions should be deliberately closed before syncing
- One canonical file should own each durable topic
- Catch-up syncs and periodic health checks are preferred over constant maintenance

## Risks

- Overengineering the system
- Creating too many folders without enough value
- Confusing temporary workspaces with long-term knowledge
- Letting duplicate records drift out of sync across modules

## Knowledge Generated

- operating principles
- module boundaries
- project lifecycle rules
- sync workflows

## Lessons Learned

- The system gets better when boundaries are explicit
- Too much ceremony creates friction and turns the system into a chore
- Dashboards should reflect real state changes, not ritual updates

## Bobby Recommendations

- Keep building only what improves judgment
- Resist cleverness that makes the system harder to use

## Codex Tasks
