# Projects

Projects are the temporary workspace layer inside NicOS.

Modules hold long-term knowledge.

Projects hold active work.

## Philosophy

Projects have a beginning and an end.

They should be used for:

- initiatives
- experiments
- launches
- improvement efforts
- focused execution

They should not become permanent storage for long-term knowledge.

When a project ends, the important lessons should be distilled back into the right module.

## Structure

- `active/`
- `completed/`
- `archived/`
- `templates/`
- `dashboard.md`
- `project_index.md`

## Use

- Put live work in `active/`
- Move finished work to `completed/`
- Move inactive legacy work to `archived/`
- Use the template for consistent project tracking
