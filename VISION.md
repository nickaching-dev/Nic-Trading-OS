# Vision

The purpose of NicOS is to become a personalized operating system for Nic's life, learning, projects, and decision-making.

It should help Nic become more thoughtful, more disciplined, and more consistent across the areas that matter most.

# Mission

Capture every meaningful lesson, chart, trade, project, and rule so that improvement compounds over time.

The system should help Nic:

- Think clearly.
- Reduce emotional decisions.
- Review mistakes honestly.
- Build confidence through process rather than outcomes.

# Core Principles

1. Process over profit.
2. Protect downside before chasing upside.
3. Wait for confirmation when evidence matters.
4. Simplicity beats complexity.
5. Every meaningful action should teach something.
6. Rules evolve only after evidence.
7. Long-term consistency is more important than short-term wins.

# Bobby's Promise

NicOS will never be optimized for making the most activity.

It will be optimized for making the best decisions.

If a feature increases complexity without improving decision quality, it should not be added.

The goal is not to chase weekly wins for their own sake.

The goal is to become a better decision-maker every year.

# What NicOS Is

A personal operating system for life, learning, projects, and judgment.

It should:

- Store context across different parts of Nic's life.
- Store rules, templates, and lessons.
- Keep execution records separate from planning notes.
- Preserve useful thinking so it can be reused.
- Help identify repeated strengths and repeated mistakes.
- Support structured review over impulsive reaction.

# The Three Layers

## System

The system defines how NicOS works.

It holds the operating philosophy, rules, lifecycle logic, and coordination patterns.

## Modules

Modules are permanent knowledge domains.

They should preserve long-term knowledge in areas such as:

- Trading
- Investing
- Restaurant
- Business
- Reflection
- Health
- Learning

## Projects

Projects are temporary initiatives with a beginning and an end.

They are workspaces, not permanent knowledge repositories.

Projects should create:

- experiments
- plans
- execution work
- decisions
- lessons

When a project is finished, the useful knowledge it generates should be distilled back into the right module.

# Modules

NicOS should grow through focused modules rather than one giant system.

Module status today:

- Trading: active
- Investing: initialized placeholder until durable investing knowledge is synced
- Restaurant: initialized with templates
- Reflection: initialized with templates
- Business: placeholder
- Health: placeholder
- Learning: placeholder

# The Project Layer

Projects give NicOS motion.

Modules store the long-term knowledge.

Projects create the short-term action and focused effort.

The relationship should be:

Modules guide projects.

Projects generate knowledge.

Knowledge returns to modules.

# What The Trading Module Is

The trading module is a personal operating system for swing trading.

It should:

- Store trading context.
- Store watchlists.
- Store trading rules.
- Store trade journals.
- Store trading lessons.
- Store chart analyses.
- Generate meaningful review statistics over time.

# Boundary

Betting is intentionally excluded from NicOS and is managed separately.

The separate `Bobby | Betting` conversation may continue outside NicOS, but it must not sync into the repository.

# What NicOS Is NOT

- A signal service.
- A stock picker.
- A prediction engine.
- A replacement for critical thinking.
- A system that rewards activity for its own sake.

# Long-Term Goal

Become a second brain that understands how Nic thinks, acts, learns, and improves across domains through data, reflection, and structured learning.
